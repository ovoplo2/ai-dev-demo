# PLANS.md

## Project
EU/US mobile web crystal + zodiac bracelet builder

## Planning Rule
Work milestone by milestone.
Do not move to the next milestone until the current milestone passes its acceptance criteria and validation commands.

## Stop-and-Fix Rule
If any validation step fails, stop and repair before continuing.
Do not stack new work on top of an unstable baseline.

## Recovery Rule
If a step changes persistent data or environment state, document:
- whether it is repeatable
- how to reset it
- how to safely re-run it

## Locked Decisions
- V1 is mobile-first web, not app
- V1 must include tap-add, fly-in, drag-to-delete, auto-reflow, price/count updates, tap-to-replace
- V1 must not include free-form reorder drag
- visual language must not resemble prior AI/SaaS site style
- prioritize simple stable implementations over flashy but fragile ones
- WooCommerce integration is delayed until builder + save/load is stable

## Milestone 1 - Builder interaction shell

### Goal
Ship a mobile builder shell with the locked V1 interactions working locally.

### Scope
- builder page structure
- bracelet preview
- tap to add
- fly-in feedback
- long-press drag-to-delete
- delete zone
- auto reflow
- real-time price/count feedback
- tap-to-replace behavior

### Acceptance Criteria
- builder page renders correctly on a mobile viewport without structural breakage
- user can tap a bead card to add a bead
- added bead appears with visible fly-in feedback
- user can long-press a bead in bracelet preview and drag it to the delete zone
- dropping a bead in the delete zone removes it
- bracelet auto-reflows after add/delete
- price updates immediately after add/delete/replace
- bead count updates immediately after add/delete/replace
- user can tap a bead in bracelet preview, then tap a catalog bead to replace it in place
- switching builder tabs does not destroy current bracelet state

### Validation Commands
```bash
npm install
npm run lint
npm run build
npm run test
```

### Manual Validation
1. Open the builder locally in a mobile viewport.
2. Add 3 beads.
3. Confirm fly-in feedback is visible for each add.
4. Select 1 bead in the bracelet.
5. Tap another bead card to replace it.
6. Confirm replacement happens in place.
7. Long-press 1 bead and drag it to the delete zone.
8. Confirm the bead is removed.
9. Confirm the bracelet reflows.
10. Confirm count and total price update correctly.

## Milestone 2 - Backend catalog + quote

### Goal
Move catalog and quote logic to backend APIs.

### Scope
- backend catalog endpoint
- backend quote endpoint
- frontend fetches catalog from API
- frontend requests quote from API
- meaning summary comes from backend rules

### Acceptance Criteria
- frontend loads bead/charm catalog from backend API
- frontend no longer depends on hardcoded catalog data for primary rendering
- quote is calculated by backend API
- meaning summary is generated by backend API
- frontend remains responsive while waiting for quote refresh
- quote response always matches current design payload format

### Validation Commands
```bash
npm install
npm run lint
npm run build
npm run test
node server.js
```

### Manual Validation
1. Start the API server.
2. Visit the builder.
3. Confirm catalog renders from API data.
4. Add or delete beads.
5. Confirm frontend calls quote endpoint.
6. Confirm total price updates from backend result.
7. Confirm meaning summary updates from backend result.

### API Validation
- `GET /api/catalog` returns valid catalog JSON
- `POST /api/quote` returns `total_price` and `meaning_summary`

## Milestone 3 - Save/load design

### Goal
Allow saving a design and restoring it later.

### Scope
- design save endpoint
- design load endpoint
- frontend save flow
- restore via query param or route param
- graceful invalid-id handling

### Acceptance Criteria
- user can save current design
- backend returns a design id
- opening the app with the design id restores the same design
- restored design preserves beads, charms, size, price, and meaning summary
- invalid or missing design id is handled gracefully without breaking the app

### Validation Commands
```bash
npm install
npm run lint
npm run build
npm run test
node server.js
```

### Manual Validation
1. Create a bracelet design.
2. Save it.
3. Copy the returned design URL or ID.
4. Reload using that design ID.
5. Confirm the same design is restored.
6. Try an invalid design ID.
7. Confirm the UI shows a safe error or fallback state.

### API Validation
- `POST /api/designs` saves design and returns `design_id`
- `GET /api/designs/:id` returns stored design JSON
